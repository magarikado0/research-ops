# docs/ — 完全版ドキュメント

STATE/ が参照する、細部まで抜けのない記録。**網羅性が正義、鮮度は二の次。原則 write-once。**
詳細はここに全文を置き、STATE には要点1行 + ここへのリンクだけを書く(single source of truth)。

この `docs/` は研究全体で共有する文書の既定位置。
サブテーマ固有の結果や考察を別フォルダに置く場合は、その場所を `research-ops.yml` の
`document_roots` に登録する。同じ内容をここへ複製せず、要約とリンクだけを置く。

サブフォルダ(実際に書くものが出たときに作る — 空フォルダを先に並べない):

- `activities/` — 実験・観察・分析・調査など、研究活動ごとの詳細記録
- `sources/` — 論文・史料・インタビューなど、参照資料ごとの完全なノート
- `materials/` — データ・試料・コーパス・参加者・対象システムなどの完全な説明
- `methods/` — 手法・手順・分析方法・実装上の注意
- `explorations/` — 捨てた方向の検討記録の完全版(`STATE/design.md` の「捨てた理由」の裏付け)
- `reviews/` — PANELなどの審議記録。人間の承認前は決定ログへ転記しない
- `notes/` — 議論・アイデアのダンプ。**未整理でよい**(整った記録を求めて記録自体をサボらないための受け口)

ルール:
- 書き込みは docs ファースト。作業した**その瞬間**に書く。
- 書いたら原則いじらない。ただし結論が覆ったら **errata の一行**を追記する
  (書き換えず打ち消す。例:「※ decisions.md 2026-08-01 で棄却(前提条件の誤りが判明)」)。
- 継続更新が要るのは `materials/` と `methods/` 程度。
