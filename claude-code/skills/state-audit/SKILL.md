# state-audit

リポジトリ直下の `OPERATIONS.md` の **§8(AUDIT手続き)** に従う。要点:

1. 現在の会話履歴を渡さない独立サブエージェントに **RESEARCH_PROFILE.md と STATE/ だけ**
   (docs や git を見せない)を読ませ、
   ①プロジェクトの説明 ②次の一手と理由 ③STATE だけでは分からなかったこと、を答えさせる。
   - 同じ会話内で「初見のつもり」になる代替は禁止する。
   - 独立実行できなければ、プロファイルとSTATEだけを渡す新規セッション用の指示文を提示して監査を停止する。
2. 返ってきた誤解と「分からなかった」リストが、そのまま STATE の欠落箇所。
3. 初見回答の確定後、メインの実行者が機械チェックする: STATE内リンク切れ / STATE要約と
   登録済み文書ルートの食い違い / ルート間の重複 / `最終確認` が古すぎる静的ファイル。
4. 検出された欠落は state-sync のdry-runで修正案を提示し、自動適用しない。
