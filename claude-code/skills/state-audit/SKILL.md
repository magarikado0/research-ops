# state-audit

リポジトリ直下の `OPERATIONS.md` の **§7(AUDIT 手続き)** に従う。要点:

1. コンテクストを持たない実行者に **STATE/ だけ**(docs や git を見せない)を読ませ、
   ①プロジェクトの説明 ②次の一手と理由 ③STATE だけでは分からなかったこと、を答えさせる。
   - 可能なら、普段 state-sync を回すのとは別系統のモデルにやらせる(自己参照ループを避ける)。
2. 返ってきた誤解と「分からなかった」リストが、そのまま STATE の欠落箇所。
3. 機械チェック: STATE 内リンク切れ / STATE要約と docs実体の食い違い / `最終確認` が古すぎる静的ファイル。
4. 検出された欠落を state-sync で補う。
