# state-sync

リポジトリ直下の `OPERATIONS.md` を読み、その **§3(蒸留)と §6(SYNC 手続き)** に厳密に従って
`STATE/` を更新する。要点(詳細は OPERATIONS.md が正):

- 差分駆動: 前回同期日以降に変わった docs / git / 実験ログだけを対象にする。
- 自己申告禁止: 記憶でなく `git log` と実験ログから逆算して書く。
- 削除も仕事: 行動に影響しなくなった項目は沈殿/削除する(完全版は docs にある)。
- 物差し: 「よい研究を進める上で**今**知っておくべきか。知らないと判断を誤りうるか」。
- `decisions.md` は追記のみ。目次腐敗(README の説明と実態のズレ)をチェックする。
- 変更は diff 形式で要約報告する。導入初期は自動適用せず、人間の承認を待つ。
