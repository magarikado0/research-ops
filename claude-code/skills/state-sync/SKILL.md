# state-sync

リポジトリ直下の `OPERATIONS.md` をすべて読み、その **§3(二段階の蒸留)と
§7(STATE SYNC手続き)** に厳密に従う。
要点(詳細は OPERATIONS.md が正):

- `last_state_sync_commit` と `research-ops.yml` の `document_roots` を先に検証する。
- 差分駆動: 前回STATE SYNC以後の登録済み文書だけを対象にする。
- 活動ログと原記録を直接STATEへ蒸留しない。完全版が欠けていれば先にdocs-syncを行う。
- 登録されたすべての文書ルートを一つの文書層として照合する。ルート間で同じ詳細を複製しない。
- 削除も仕事: 行動に影響しなくなった項目は沈殿または削除する(完全版は登録済み文書ルートにある)。
- 物差し: 「よい研究を進める上で**今**知っておくべきか。知らないと判断を誤りうるか」。
- `decisions.md` は追記のみ。目次腐敗(README の説明と実態のズレ)をチェックする。
- 既定はdry-run。変更案をdiff形式で報告し、人間が明示的に適用を指示した場合だけ更新する。
- 適用成功後に限り、STATE SYNC用のコミットSHAとタイムゾーン付き時刻を進める。
