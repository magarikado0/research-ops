# state-sync

リポジトリ直下の `OPERATIONS.md` をすべて読み、その **§3(蒸留)と §6(SYNC 手続き)** に厳密に従う。
要点(詳細は OPERATIONS.md が正):

- `STATE/README.md` のカーソルと `research-ops.yml` の `document_roots`、`activity_logs` を先に検証する。
- 差分駆動: `last_sync_commit..HEAD`、未コミット差分、登録済み活動ログのカーソル以後だけを対象にする。
- 自己申告禁止: 記憶でなく `git log`、活動ログ、原記録から逆算して書く。
- 登録されたすべての文書ルートを一つの文書層として照合する。ルート間で同じ詳細を複製しない。
- 削除も仕事: 行動に影響しなくなった項目は沈殿または削除する(完全版は登録済み文書ルートにある)。
- 物差し: 「よい研究を進める上で**今**知っておくべきか。知らないと判断を誤りうるか」。
- `decisions.md` は追記のみ。目次腐敗(README の説明と実態のズレ)をチェックする。
- 既定はdry-run。変更案をdiff形式で報告し、人間が明示的に適用を指示した場合だけ更新する。
- 適用成功後に限り、コミットSHA、タイムゾーン付き時刻、活動ログ位置のカーソルを進める。
