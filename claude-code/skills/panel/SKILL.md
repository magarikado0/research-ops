# panel

リポジトリ直下の `OPERATIONS.md` の **§8(PANEL 手続き)** に従う。`STATE/README.md`(必要なら docs)を
ブリーフにする。演劇(追従的同調による早すぎる合意)を避けるため、以下を厳守:

- **独立ドラフト → 相互反論 → 統合** の順。最初からライブ会話にしない。
- 役割は失敗モード担当(査読者2号 / データ懐疑派 / ベースライン原理主義者 / 出口担当 / 撤退判断人)。
  各役割は subagent として並列に走らせる。
- 各主張に**根拠(実験ID・論文・コード箇所)**と「覆る条件+それを確かめる実験1つ」を必須にする。
- 可能なら役割の一部を別ベンダーのモデルに割る(意見分布の均質化対策)。
- 結論は `STATE/decisions.md` に追記して閉じる。
