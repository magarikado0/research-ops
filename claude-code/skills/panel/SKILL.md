# panel

リポジトリ直下の `OPERATIONS.md` の **§8(PANEL 手続き)** に従う。`RESEARCH_PROFILE.md` に
定義された役割を使い、`STATE/README.md` と登録済み文書ルートから必要最小限の文書をブリーフにする。
演劇(追従的同調による早すぎる合意)を避けるため、以下を厳守:

- **独立ドラフト → 相互反論 → 統合** の順。最初からライブ会話にしない。
- プロファイルの各失敗モードをsubagentとして並列に走らせる。
- 各主張に**根拠(活動ID・資料・記録・コード箇所)**と「覆る条件+それを確かめる活動1つ」を必須にする。
- 可能なら役割の一部を別ベンダーのモデルに割る(意見分布の均質化対策)。
- 独立実行できなければ、1つの会話内で役割を順番に演じず、独立セッションへ配るブリーフを出力して停止する。
- 統合結果は `docs/reviews/panel-YYYYMMDD-<topic>.md` への保存案として提示する。
- `STATE/decisions.md` には自動追記しない。人間が明示承認した場合だけ、承認者・採否・
  パネル記録へのリンクを追記する。
