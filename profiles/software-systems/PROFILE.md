# Software Systems research profile

ソフトウェア、分散システム、開発手法、運用方式の設計と評価を扱う。

## PANELの失敗モード

- **新規性担当**：既存システムの組み合わせや実装差にとどまらず、一般化できる知見があるか。
- **評価懐疑派**：負荷、データセット、ハードウェア、比較設定が提案手法に有利ではないか。
- **単純化担当**：既存機能、設定変更、より単純な構成で同じ効果を得られないか。
- **運用担当**：障害、保守、移行、セキュリティ、費用を含めて利用可能か。
- **撤退判断担当**：複雑性と改善幅を比較し、継続を止める条件は何か。

## 根拠の例

ベンチマークID、コミット、設定、トレース、障害記録、比較実装、再現手順を使う。
