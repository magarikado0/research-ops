1. コンテクストを持たない読み手になったつもりで、STATE/ だけ(docs や git を見ない)を読み、
   ①プロジェクトの説明 ②次の一手と理由 ③STATE だけでは分からなかったこと、を答えよ。
2. ②③の誤解と「分からなかった」項目が、そのまま STATE の欠落箇所である。列挙せよ。
3. 機械チェック: STATE 内リンク切れ / STATE要約と docs実体の食い違い / `最終確認` が古すぎる静的ファイル。
4. 検出された欠落を、続けて state-sync の手順で補え(または補うべき差分を提案せよ)。

※ この監査は、普段 state-sync を回すのとは別系統のモデルで実行すると自己参照バイアスを避けられる。
