# panel

リポジトリ直下の `OPERATIONS.md` の §9 に従う。`RESEARCH_PROFILE.md` に定義された役割を使い、
`STATE/README.md` と、登録済み文書ルートから必要最小限の文書をブリーフにする。

1. プロファイルにある各失敗モードを、互いの出力を見せない独立サブエージェントとして並列実行する。
2. 各主張に活動ID・資料・記録・コード箇所などの根拠、覆る条件、確認活動を要求する。
3. 第2ラウンドで他者のメモを渡し、同意できない点だけを具体的に反論させる。
4. 独立実行を利用できない場合、1つの会話内で役割を順番に演じて代替しない。独立セッションへ
   配布できるブリーフを出力して停止する。
5. 統合結果は `docs/reviews/panel-YYYYMMDD-<topic>.md` への保存案として提示する。
6. `STATE/decisions.md` には自動追記しない。人間が結論を明示承認した場合だけ、承認者、採否、
   パネル記録へのリンクを追記する。
