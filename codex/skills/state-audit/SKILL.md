# state-audit

リポジトリ直下の `OPERATIONS.md` の §7 に従う。

1. 現在の会話履歴を渡さない独立サブエージェントを起動し、`RESEARCH_PROFILE.md` と `STATE/` 以外を読まないよう明示する。
2. その読み手に、目的・現在地・アプローチ、次の一手と理由、不明点を回答させる。
3. 独立実行が利用できない場合は「初見のつもり」で代替せず、STATEだけを渡す新規セッション用の
   指示文と `RESEARCH_PROFILE.md` を提示して監査を停止する。
4. 初見回答を確定した後、メインの実行者がリンク切れ、登録済み文書ルートとの不一致、
   ルート間の重複、古い最終確認日を検査する。
5. 欠落の修正は `$state-sync` と同じdry-run規約で差分提案まで行い、自動適用しない。
